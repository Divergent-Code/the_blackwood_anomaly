# API Reference: The Blackwood Anomaly

The Blackwood Anomaly provides a RESTful API powered by FastAPI. This document outlines the available endpoints, data models, and the authentication mechanism required to interact with the game engine.

## Authentication

The API uses a **Bring-Your-Own-Key (BYOK)** architecture. All game endpoints require a valid **Gemini, OpenAI, or OpenRouter API Key** to be passed via the `Authorization` header using the Bearer schema.

This ensures that the server remains stateless regarding API billing, and each request utilizes an isolated GenAI client.

**Header Format:**

```http
Authorization: Bearer <YOUR_API_KEY>
X-LLM-Provider: gemini
X-GM-Model: gemini-2.5-flash
X-Mechanics-Model: gemini-2.5-flash
```

*Note: `X-LLM-Provider` defaults to `gemini`, but can be set to `openai` or `openrouter` depending on your preferred API. `X-GM-Model` sets the primary storytelling model, while `X-Mechanics-Model` is used for tool calling and logic.*

## Data Models

### `ActionRequest`

Payload sent when a player performs an action in the game.

| Field    | Type     | Description                                      |
| -------- | -------- | ------------------------------------------------ |
| `action` | `string` | The text describing what the player wants to do. |

### `GameResponse`

Payload returned after initializing a session or processing an action. Contains the updated game state and the narrative response from the AI Game Master.

| Field              | Type      | Description                                                  |
| ------------------ | --------- | ------------------------------------------------------------ |
| `session_id`       | `string`  | The UUID of the current game session.                        |
| `health`           | `integer` | The player's current health (0-100).                         |
| `stress`           | `integer` | The player's current stress level (0-100).                   |
| `narrative`        | `string`  | The descriptive atmospheric text generated by the AI.        |
| `agent_actions`    | `array`   | Array of strings detailing any agentic tool calls.           |
| `inventory`        | `array`   | Array of items currently in the player's possession (max 4). |
| `current_location` | `string`  | The player's current canonical location.                     |
| `discovered_lore`  | `array`   | Array of lore fragment IDs the player has found.             |
| `escape_stage`     | `integer` | The current stage of the escape route (0-4).                 |
| `turn_count`       | `integer` | The number of actions the player has taken.                  |
| `game_over`        | `boolean` | True if health is <= 0.                                      |
| `game_won`         | `boolean` | True if escape_stage is >= 4.                                |
| `panic_cascade`    | `boolean` | True if stress is >= 80.                                     |

### `SessionState`

Payload representing a saved game session.

| Field              | Type      | Description                                           |
| ------------------ | --------- | ----------------------------------------------------- |
| `session_id`       | `string`  | The UUID of the game session.                         |
| `health`           | `integer` | The player's current health.                          |
| `stress`           | `integer` | The player's current stress level.                    |
| `history`          | `array`   | Array of previous chat history objects.               |
| `inventory`        | `array`   | Array of items currently in the player's possession.  |
| `current_location` | `string`  | The player's current canonical location.              |
| `discovered_lore`  | `array`   | Array of lore fragment IDs the player has found.      |
| `escape_stage`     | `integer` | The current stage of the escape route (0-4).          |
| `turn_count`       | `integer` | The number of actions the player has taken.           |

### `RecapResponse`

Payload returned by the session recap endpoint.

| Field    | Type     | Description                                                       |
| -------- | -------- | ----------------------------------------------------------------- |
| `recap`  | `string` | A 2–3 sentence clinical atmospheric summary of the prior session. |

## Endpoints

### 1. Serve Frontend

**GET** `/`

Serves the static frontend web interface.

* **Response:** HTML File (`static/index.html`)

---

### 2. Initialize New Session

**POST** `/api/v1/sessions`

Creates a new game session, initializes the player's vitals (Health: 100, Stress: 0), and generates the opening narrative of the game.

* **Security:** `HTTPBearer` (API Key required)
* **Request Body:** None
* **Success Response:**
  * **Code:** `200 OK`
  * **Content:** [`GameResponse`](#gameresponse)
* **Error Responses:**
  * **Code:** `401 Unauthorized` (Missing or invalid API Key)
  * **Code:** `500 Internal Server Error` (AI Engine error during generation)

**Example Response:**

```json
{
  "session_id": "c9b1e9c2-5a2a-4f4b-8d19-d2b3c4d5e6f7",
  "health": 100,
  "stress": 0,
  "narrative": "You awaken on a cold, rusted metal table. The air smells of ozone and old blood...",
  "agent_actions": [],
  "inventory": [],
  "current_location": "Intake Bay 4",
  "discovered_lore": [],
  "escape_stage": 0,
  "turn_count": 0,
  "game_over": false,
  "game_won": false,
  "panic_cascade": false
}
```

---

### 3. Generate Session Recap

**POST** `/api/v1/sessions/{session_id}/recap`

Generates an atmospheric summary of a prior play session by compressing the full conversation history into a labelled transcript and invoking the LLM in **MODE 2 (Session Recap)**. Designed for returning players — the frontend calls this immediately after restoring session state.

> **Note:** The LLM is explicitly instructed **not** to append the `[Health: X% | Stress: Y%]` vitals tag in this mode, preventing the regex extractor in `submit_action` from misfiring on recap text.

* **Security:** `HTTPBearer` (API Key required)
* **Path Parameters:**
  * `session_id` (`string`): The UUID of the session to recap.
* **Request Body:** None
* **Success Response:**
  * **Code:** `200 OK`
  * **Content:** [`RecapResponse`](#recapresponse)
* **Error Responses:**
  * **Code:** `401 Unauthorized` (Missing or invalid API Key)
  * **Code:** `404 Not Found` (Session ID does not exist)
  * **Code:** `500 Internal Server Error` (LLM recap generation failed)

**Example Response:**

```json
{
  "recap": "Subject 814 regained consciousness in Corridor 7-B, secured a scalpel from the surgical bay, and barricaded the west fire door before blood loss forced a retreat. They now stand at the junction, the east corridor still unexplored. What do you do?"
}
```

### 4. Submit Action

**POST** `/api/v1/sessions/{session_id}/actions`

Processes a player's submitted action within an existing session. The engine retrieves relevant game rules via RAG, updates the system prompt with the player's current state, and invokes the LLM to generate the outcome and compute vital changes.

* **Security:** `HTTPBearer` (API Key required)
* **Path Parameters:**
  * `session_id` (`string`): The UUID of the session to update.
* **Request Body:** [`ActionRequest`](#actionrequest)
* **Success Response:**
  * **Code:** `200 OK`
  * **Content:** [`GameResponse`](#gameresponse)
* **Error Responses:**
  * **Code:** `401 Unauthorized` (Missing or invalid API Key)
  * **Code:** `404 Not Found` (Session ID does not exist in the database)
  * **Code:** `500 Internal Server Error` (AI Engine/Parsing error)

**Example Request:**

```json
{
  "action": "I cautiously search the medical cabinets for bandages."
}
```

**Example Response:**

```json
{
  "session_id": "c9b1e9c2-5a2a-4f4b-8d19-d2b3c4d5e6f7",
  "health": 100,
  "stress": 5,
  "narrative": "You pry open the rusted cabinet door. A swarm of bloated, pale centipedes spills out over your hands...",
  "agent_actions": [
    "⚙️ roll_d20(DC=15) → Roll: 4 vs DC 15. Success: False",
    "⚙️ apply_vitals(Δhealth=0, Δstress=5) → Vitals updated — Health: 100%, Stress: 5%"
  ],
  "inventory": [],
  "current_location": "Intake Bay 4",
  "discovered_lore": [],
  "escape_stage": 0,
  "turn_count": 1,
  "game_over": false,
  "game_won": false,
  "panic_cascade": false
}
```

---

### 5. List Sessions

**GET** `/api/v1/sessions`

Lists all persisted sessions.

* **Security:** None (Public Endpoint)
* **Request Body:** None
* **Success Response:**
  * **Code:** `200 OK`
  * **Content:** Array of [`SessionState`](#sessionstate)

---

### 6. Get Session by ID

**GET** `/api/v1/sessions/{session_id}`

Loads a saved session by ID — used by the frontend to resume a game.

* **Security:** None (Public Endpoint)
* **Path Parameters:**
  * `session_id` (`string`): The UUID of the session to fetch.
* **Request Body:** None
* **Success Response:**
  * **Code:** `200 OK`
  * **Content:** [`SessionState`](#sessionstate)
* **Error Responses:**
  * **Code:** `404 Not Found` (Session ID does not exist)
