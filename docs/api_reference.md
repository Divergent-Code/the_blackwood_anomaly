# API Reference: The Blackwood Anomaly

The Blackwood Anomaly provides a RESTful API powered by FastAPI. This document outlines the available endpoints, data models, and the authentication mechanism required to interact with the game engine.

## Authentication

The API uses a **Bring-Your-Own-Key (BYOK)** architecture. All game endpoints require a valid **Gemini, OpenAI, or OpenRouter API Key** to be passed via the `Authorization` header using the Bearer schema.

This ensures that the server remains stateless regarding API billing, and each request utilizes an isolated GenAI client.

**Header Format:**

```http
Authorization: Bearer <YOUR_API_KEY>
X-LLM-Provider: gemini
```

*Note: `X-LLM-Provider` defaults to `gemini`, but can be set to `openai` or `openrouter` depending on your preferred API.*

## Data Models

### `ActionRequest`

Payload sent when a player performs an action in the game.

| Field    | Type     | Description                                      |
| -------- | -------- | ------------------------------------------------ |
| `action` | `string` | The text describing what the player wants to do. |

### `GameResponse`

Payload returned after initializing a session or processing an action. Contains the updated game state and the narrative response from the AI Game Master.

| Field           | Type      | Description                                           |
| --------------- | --------- | ----------------------------------------------------- |
| `session_id`    | `string`  | The UUID of the current game session.                 |
| `health`        | `integer` | The player's current health (0-100).                  |
| `stress`        | `integer` | The player's current stress level (0-100).            |
| `narrative`     | `string`  | The descriptive atmospheric text generated by the AI. |
| `agent_actions` | `array`   | Array of strings detailing any agentic tool calls.    |

## Endpoints

### 1. Serve Frontend

**GET** `/`

Serves the static frontend web interface.

* **Response:** HTML File (`static/index.html`)

---

### 2. Initialize New Session

**POST** `/api/v1/sessions`

Creates a new game session, initializes the player's vitals (Health: 100, Stress: 0), and generates the opening narrative of the game.

* **Security:** `HTTPBearer` (API Key required)
* **Request Body:** None
* **Success Response:**
  * **Code:** `200 OK`
  * **Content:** [`GameResponse`](#gameresponse)
* **Error Responses:**
  * **Code:** `401 Unauthorized` (Missing or invalid API Key)
  * **Code:** `500 Internal Server Error` (AI Engine error during generation)

**Example Response:**

```json
{
  "session_id": "c9b1e9c2-5a2a-4f4b-8d19-d2b3c4d5e6f7",
  "health": 100,
  "stress": 0,
  "narrative": "You awaken on a cold, rusted metal table. The air smells of ozone and old blood...",
  "agent_actions": []
}
```

---

### 3. Submit Action

**POST** `/api/v1/sessions/{session_id}/actions`

Processes a player's submitted action within an existing session. The engine retrieves relevant game rules via RAG, updates the system prompt with the player's current state, and invokes the LLM to generate the outcome and compute vital changes.

* **Security:** `HTTPBearer` (API Key required)
* **Path Parameters:**
  * `session_id` (`string`): The UUID of the session to update.
* **Request Body:** [`ActionRequest`](#actionrequest)
* **Success Response:**
  * **Code:** `200 OK`
  * **Content:** [`GameResponse`](#gameresponse)
* **Error Responses:**
  * **Code:** `401 Unauthorized` (Missing or invalid API Key)
  * **Code:** `404 Not Found` (Session ID does not exist in the database)
  * **Code:** `500 Internal Server Error` (AI Engine/Parsing error)

**Example Request:**

```json
{
  "action": "I cautiously search the medical cabinets for bandages."
}
```

**Example Response:**

```json
{
  "session_id": "c9b1e9c2-5a2a-4f4b-8d19-d2b3c4d5e6f7",
  "health": 100,
  "stress": 5,
  "narrative": "You pry open the rusted cabinet door. A swarm of bloated, pale centipedes spills out over your hands... [Health: 100% | Stress: 5%]",
  "agent_actions": [
    "⚙️ Planner Agent decided to roll_d20(DC=15) -> Roll: 4 vs DC 15. Success: False"
  ]
}
```
